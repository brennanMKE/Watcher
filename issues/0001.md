## Description

The public types `Session`, `Event`, and `Options` were declared at the top level of the `Watcher` module. `Session` in particular is too generic — it collides with `Session` types from other frameworks when imported in the same file, and tells the reader nothing about what kind of session it is at the call site. The existing `Watcher` caseless enum was a one-property holder for `logSubsystem` rather than a real namespace.

## Root cause

Top-level public symbols competed with the module's bare namespace instead of being consolidated under it. Modern Swift idiom is to nest related public types under a shared namespace (`URLSession.DataTask`, `URLSession.Configuration`, `Combine.Publishers.Map`), and the package was missing that consolidation.

## Fix

Wrapped `Session`, `Event`, and `Options` in `extension Watcher { ... }`, so they're now spelled `Watcher.Session`, `Watcher.Event`, and `Watcher.Options` at call sites. `WatcherError` stays top-level by design — it's already self-naming and external code may reference it directly. The internal `FolderEngine.init` was updated to take `Watcher.Options.Scope` / `Watcher.Options.Depth`. Tests and README updated to match.

## Files changed

- `Sources/Watcher/Event.swift` — wrapped `Event` in `extension Watcher`.
- `Sources/Watcher/Options.swift` — wrapped `Options` (with nested `Scope` / `Depth`) in `extension Watcher`.
- `Sources/Watcher/Session.swift` — wrapped `Session` in `extension Watcher`; updated the `EngineOwner` conformance extension to `extension Watcher.Session`.
- `Sources/Watcher/Internal/FolderEngine.swift` — qualified the `scope` / `depth` init parameters as `Watcher.Options.Scope` / `Watcher.Options.Depth`.
- `Tests/WatcherTests/WatcherTests.swift`, `FileWatchTests.swift`, `FolderWatchTests.swift`, `HardeningTests.swift` — updated all references to the nested spellings.
- `README.md` — updated the public API table and usage example.

## Gotchas

- Inside `extension Watcher { ... }`, sibling nested types resolve by their bare name (`Session` can refer to `Event` without qualification). External callers must use the fully qualified name (`Watcher.Event`).
- `WatcherError` was deliberately not nested. Renaming it to `Watcher.Error` was considered but skipped — it's a stable public type that external code may already reference, and the `WatcherError` name is already self-describing.
- `PRD.md` and `Concepts.md` still reference the bare type names. They were not updated as part of this change and remain a follow-up doc pass.
- DocC references like `` ``Session`` `` inside doc comments continue to resolve because they're inside the same enclosing scope after the nest. They could be rewritten as `` ``Watcher/Session`` `` for clarity in generated docs but were left as-is.
